В папке tests находятся ваши тесты + скомпилированный exe-файл + мой тест (проверяет второе условие, вы его в своих тестах почему-то не проверили, т.е. он проверяет ситуацию, когда на столе уже 25 карт, но явных нарушений не было и колода не закончилась)

В папке hanabi лежит 3 проекта:
	1) hanabi - Основной проект, там каждый класс в своём файле + в классе Program есть удобная (на мой взгляд) система дебиггинга
	2) hanabi.tester - Собственно, ваши тесты + мой тест. Использовался NUnit версии 3.0.1. Чтобы он работал корректно, необходимо папку tests положить по пути C:\tests\ (не получилось нормально подружить NUnit и ReSharper)
	3) musin_rustam - Проект, где лежит один файл Program.cs. В нём собраны все классы вместе. По сути является почти полной копией классов из проекта (1), за исключением средств для дебаггинга
